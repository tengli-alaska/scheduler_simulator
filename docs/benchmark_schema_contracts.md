# Benchmark Schema Contracts

This document defines the canonical data contracts used by the benchmark tool.

## 1) Suite Specification

- Location: `benchmark/spec/suites/*.json`
- Schema: `benchmark/spec/schema/benchmark_suite.schema.json`
- Required `schema_version`: `suite.v1`

Required top-level fields:

- `schema_version`
- `benchmark_version`
- `suite_id`
- `title`
- `description`
- `defaults`
- `experiments`

## 2) Run Artifacts

Canonical run layout:

- `runs/<suite_id>/<run_id>/metrics.csv`
- `runs/<suite_id>/<run_id>/tasks.csv`
- `runs/<suite_id>/<run_id>/manifest.json`

Every metrics/tasks row must include provenance fields:

- `BenchmarkVersion`
- `SchemaVersion`
- `SuiteId`
- `RunId`
- `Seed`

Current schema tags:

- Metrics rows: `metrics.v2`
- Task rows: `tasks.v2`

## 3) Manifest

Manifest files are generated per run at `runs/<suite_id>/<run_id>/manifest.json`.

Manifest contents:

- Source file paths and SHA-256 hashes
- Benchmark metadata (`benchmark_version`, `suite_id`, `run_id`)
- Run parameter summary (`num_tasks`, `cores`, `stop_time`, `topology`, etc.)
- Runtime provenance (`generated_at_utc`, git commit when available)

## 4) Analysis Outputs

Produced by `scripts/aggregate_results.py`:

- `analysis/metrics_enriched.csv`
- `analysis/run_index.csv`
- `analysis/quality_checks.csv`

Produced by `scripts/export_summary_table.py`:

- `analysis/summary_table.csv`
- `analysis/summary_table.md`

`metrics_enriched.csv` includes:

- Provenance fields
- Canonical run config fields (`NumTasks`, `Cores`, `Topology`, `Balancer`, `WorkStealing`, `StopTime`)
- Experiment membership (`ExperimentIds`, plus `Exp1..Exp6` flags)
- Performance/fairness/overhead metrics and derived per-task rates

`run_index.csv` includes:

- One row per discovered metrics artifact
- Data lineage (`MetricsPath`, `TasksPath`, `ManifestPath`)
- Runner provenance (`ExecutionSummaryPath`, `ExecutionStatus`, `ExecutionLogPath`)
- Scheduler/workload/config coverage summaries

`quality_checks.csv` includes:

- `missing_cell`
- `duplicate_config`
- `low_completion_ratio`
- `invalid_log_scale_candidate`

## 5) Figures and Report

Experiment figures are generated by:

- `python3 scripts/plot_experiment_suite.py --analysis-dir analysis --output-dir figures/experiments/<suite_id>`

Optional publication/report pack:

- `python3 -m benchmark.report --suite <suite_id_or_path> --format md`

Outputs:

- `figures/publication/<suite_id>/`
- `reports/<suite_id>/report.md`

## 6) Plugin Registration

Suite scheduler/workload keys resolve through the plugin registry.

Core types:

- `SchedulerPlugin(key, cli_token, display_name, aliases=..., is_meta=...)`
- `WorkloadPlugin(key, cli_token, display_name, aliases=..., is_meta=...)`

Required plugin module interface:

```python
def register(registry) -> None:
    ...
```

Discovery order:

- Built-ins (`benchmark/plugins/builtins.py`)
- Local plugins (`benchmark/plugins/local/*.py`)
- Optional external paths via `--plugins` / `BENCHMARK_PLUGIN_PATHS`
